## 一、指针基础

### 1. 核心概念

1. **内存地址**：计算机内存的每个字节都有唯一编号，这个编号就是**地址**。
2. **指针**：指针的本质就是**内存地址**。
3. **指针变量**：专门用来**存储内存地址**的变量（普通变量存数值，指针变量存地址）。

### 2. 两个核心运算符

| 运算符 | 名称     | 作用                     | 示例                     |
| ------ | -------- | ------------------------ | ------------------------ |
| `&`    | 取地址符 | 获取变量的内存地址       | `&a` → 取变量`a`的地址   |
| `*`    | 解引用符 | 通过地址访问对应内存的值 | `*p` → 取指针`p`指向的值 |

### 3. 指针定义语法

```
// 格式：数据类型 *指针变量名;
int a = 10;
int *p;      // 定义int类型指针（只能存储int变量的地址）
p = &a;      // 把a的地址赋值给指针p
*p = 20;     // 解引用：修改p指向的内存（a的值变成20）
```

关键：**指针的类型必须和指向变量的类型一致**。

## 二、指针的基本特性

### 1. 空指针

- 定义：指向**地址 0**的指针，代表「不指向任何有效内存」。
- 用法：初始化指针，避免野指针。

```
int *p = NULL;  // NULL是标准定义的空指针常量
```

禁止：**空指针解引用**（`*NULL`）会直接导致程序崩溃！

### 2. 野指针

- 定义：指向**非法 / 已释放内存**的指针（最常见的 bug 来源）。
- 产生原因：指针未初始化、指向局部变量、内存已释放。
- 规避：指针必须初始化，释放内存后置`NULL`。

### 3. 指针的步长

指针`++/--`时，移动的字节数 = 指向数据类型的大小：

- `int*` → 步长 4 字节
- `char*` → 步长 1 字节
- `float*` → 步长 4 字节

作用：指针遍历数组时自动跳转到下一个元素。

## 三、指针与数组

### 1. 核心结论

**数组名 = 数组首元素的地址**（常量指针，不能修改指向）。

```
int arr[5] = {1,2,3,4,5};
int *p = arr;  // 等价 int *p = &arr[0];
```

### 2. 指针访问数组

数组下标 和 指针写法**完全等价**：

| 数组写法  | 指针写法 | 含义              |
| --------- | -------- | ----------------- |
| `arr[i]`  | `*(p+i)` | 第 i 个元素的值   |
| `&arr[i]` | `p+i`    | 第 i 个元素的地址 |

### 3. 指针遍历数组

{% raw %}

```
int arr[5] = {1,2,3,4,5};
int *p = arr;
for(int i=0; i<5; i++){
    printf("%d ", *p++);  // p++：指向下一个元素
}
```

{% endraw %}

## 四、指针与字符串

C 语言字符串 = 以`\0`结尾的`char`数组，字符串操作全靠指针。

### 1. 两种定义方式

```
// 1. 字符数组（栈区，内容可修改）
char str1[] = "hello"; 
str1[0] = 'H';  //合法

// 2. 字符指针（常量区，内容不可修改）
char *str2 = "hello";
str2[0] = 'H';  //报错！只读内存
```

### 2. 指针操作字符串

{% raw %}

```
char *p = "hello";
while(*p != '\0'){  // 遍历到字符串结束符
    printf("%c", *p++);
}
```

{% endraw %}

## 五、指针与函数（传址调用）

### 1. 传值 vs 传址

1. **传值**：函数内修改形参，**不影响实参**。
2. **传址**：用指针传递地址，函数内**直接修改实参**。

### 2. 交换两个数（传址经典案例）

{% raw %}

```
void swap(int *a, int *b){
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main(){
    int x=1, y=2;
    swap(&x, &y);  // 传递地址
    printf("%d %d", x,y); // 输出2 1
    return 0;
}
```

{% endraw %}

### 3.指针作为函数返回值

禁止：返回局部变量的地址（函数结束后栈内存释放，变成野指针）。

允许：返回全局变量、静态变量、动态分配内存的地址。

## 六、指针进阶

### 1. 多级指针（指针的指针）

存储「一级指针的地址」，用于修改指针本身

```
int a = 10;
int *p = &a;    // 一级指针
int **pp = &p;  // 二级指针：指向p的地址
**pp = 20;      // 等价 *p = 20 → a=20
```

### 2. 数组指针 vs 指针数组

| 写法          | 名称     | 含义                               |
| ------------- | -------- | ---------------------------------- |
| `int (*p)[5]` | 数组指针 | 指向【长度为 5 的 int 数组】的指针 |
| `int *p[5]`   | 指针数组 | 数组，元素是 5 个 int 指针         |

### 3. 函数指针

指向**函数**的指针，存储函数的入口地址。

```
// 定义：返回值类型 (*指针名)(参数列表)
int add(int a,int b){ return a+b; }
int (*fp)(int,int) = add;  // fp指向add函数
int res = fp(1,2);  // 等价 add(1,2)
```

### 4. 回调函数

把**函数指针作为参数**传给另一个函数，实现灵活调用。

## 七、指针与结构体

结构体指针访问成员用 `->` 运算符（比`.`更高效）。

{% raw %}

```
struct Student{
    int age;
    char name[10];
};
int main(){
    struct Student s = {18, "张三"};
    struct Student *p = &s;
    printf("%d %s", p->age, p->name);  // 指针访问成员
    return 0;
}
```

{% endraw %}

## 八、指针与动态内存

用指针管理**堆区内存**（手动申请 / 释放，解决数组固定大小问题），头文件：`<stdlib.h>`。

### 1. 核心函数

| 函数      | 作用                    | 示例                               |
| --------- | ----------------------- | ---------------------------------- |
| `malloc`  | 申请未初始化的内存      | `int *p = malloc(5*sizeof(int));`  |
| `calloc`  | 申请并初始化为 0 的内存 | `int *p = calloc(5, sizeof(int));` |
| `realloc` | 调整已申请内存的大小    | `p = realloc(p, 10*sizeof(int));`  |
| `free`    | 释放内存                | `free(p); p=NULL;`                 |

### 2. 规范写法

{% raw %}

```
int *p = (int*)malloc(5 * sizeof(int));
if(p == NULL){  // 必须判断：内存申请失败
    exit(1);
}
// 使用内存...
free(p);  // 释放
p = NULL; // 避免野指针
```

{% endraw %}

### 3.申请它做什么？

这部分其实很多时候都不会讲，再者就是浅浅一过。但是实际做项目就知道这块很重要。

为什么要申请内存？它的作用是什么？其实说白了内存就是个仓库，我们就是租仓库放东西。

有人会说数组也可以放东西啊，为什么要内存呢？因为数组在我们程序编译的时候大小就已经定好了，但有时候我们要存的数据大小不一定是多少，小于尺寸还好说，但是一旦大了那就不太好解决了。这时候我们申请一块内存，可以把它当作可变长数组。

这时候就有人又会说我把数组定大一点不就行了，是可以，但是远没有申请内存能够获得更多。数组最大也就占满栈(Stack)空间然后溢出，相比内存申请这种堆(Heap)不是一个数量级差别。而且我们的栈要用在更重要的上面。

其实举个不恰当例子，完全可以看作是M2固态和机械硬盘这种差别。

比较需要注意的是申请完内存他是不会手动释放的。所以再也不用的时候需要手动释放，别占着内存浪费。
