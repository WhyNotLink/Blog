## 验证引导模式

要验证系统目前的引导模式，请检查 [UEFI 固件位数](https://wiki.archlinuxcn.org/wiki/UEFI#UEFI_固件位数)：

```
# cat /sys/firmware/efi/fw_platform_size
```

- 如果命令结果为 `64`，则系统是以 UEFI 模式引导且使用 64 位 x64 UEFI。
- 如果命令结果为 `32`，则系统是以 UEFI 模式引导且使用 32 位 IA32 UEFI，虽然其受支持，但引导加载程序只能使用 [systemd-boot](https://wiki.archlinuxcn.org/wiki/Systemd-boot) 和 [GRUB](https://wiki.archlinuxcn.org/wiki/GRUB)。
- 如果命令结果为`No such file or directory`，则系统可能是以 [BIOS](https://zh.wikipedia.org/wiki/BIOS) 模式（或 [CSM](https://en.wikipedia.org/wiki/Compatibility_Support_Module) 模式，这两种模式通常出现在**老旧**的电脑或**未经配置的虚拟机**上）引导。

如果系统没有以您想要的模式（UEFI 或 BIOS）引导启动，请您参考自己的计算机或主板说明书。

## 验证网络

确保系统已经列出并启用了[网络接口](https://wiki.archlinuxcn.org/wiki/网络配置#网络接口)，用 [ip-link(8)](https://man.archlinux.org/man/ip-link.8) 检查：

```
# ip link
```

对于无线局域网（Wi-Fi）和无线广域网（WWAN），请确保网卡未被 [rfkill](https://wiki.archlinuxcn.org/wiki/Rfkill) 禁用。

连接到网络：

- 有线以太网——连接网线。
- WiFi——使用 [iwctl](https://wiki.archlinuxcn.org/wiki/Iwd#iwctl) 认证无线网络。
- 移动宽带调制解调器（移动网卡） - 使用 [mmcli](https://wiki.archlinuxcn.org/wzh/index.php?title=Mmcli&action=edit&redlink=1) 连接到移动网络。

配置网络连接：

- [DHCP](https://wiki.archlinuxcn.org/wiki/网络配置#DHCP)：对于有线以太网、无线局域网（WLAN）和无线广域网（WWAN）网络接口来说，动态 IP 地址和 DNS 服务器分配（由 [systemd-networkd](https://wiki.archlinuxcn.org/wiki/Systemd-networkd) 和 [systemd-resolved](https://wiki.archlinuxcn.org/wiki/Systemd-resolved) 提供功能）能够开箱即用。
- 静态 IP 地址：按照[网络配置#静态 IP 地址](https://wiki.archlinuxcn.org/wiki/网络配置#静态_IP_地址)进行操作。

用 [ping](https://wiki.archlinuxcn.org/wiki/网络配置#Ping) 检查网络连接：

```
# ping ping.archlinux.org
```

## 创建硬盘分区

系统如果识别到计算机的内置硬盘、U盘或者移动硬盘等类型磁盘，就会将其分配为一个[块设备](https://wiki.archlinuxcn.org/wiki/设备文件#块设备)，如 `/dev/sda`、`/dev/nvme0n1` 或 `/dev/mmcblk0`。可以使用 [lsblk](https://wiki.archlinuxcn.org/wiki/Lsblk) 或者 [fdisk](https://wiki.archlinuxcn.org/wiki/Fdisk) 等工具查看：

```
# fdisk -l	#此处为小写字母l
```

结果中以 `rom`、`loop` 或者 `airootfs` 结尾的设备可以被忽略。结果中以 `rpmb`、`boot0` 或者 `boot1` 结尾的 `mmcblk*` 设备也可以被忽略。

对于一个选定的设备，以下分区是**必需**的：

- 一个根分区（挂载在[根目录](https://wiki.archlinuxcn.org/wiki/分区#根分区)）`/`；
- 要在 [UEFI](https://wiki.archlinuxcn.org/wiki/Unified_Extensible_Firmware_Interface) 模式中启动，还需要一个 [EFI 系统分区](https://wiki.archlinuxcn.org/wiki/EFI_系统分区)。

请使用分区工具（[fdisk](https://wiki.archlinuxcn.org/wiki/Fdisk) 、[parted](https://wiki.archlinuxcn.org/wiki/Parted)、[cfdisk](https://wiki.archlinuxcn.org/wiki/Cfdisk) 等等）修改分区表。例如：

```
# cfdisk /dev/要被分区的磁盘
```

| 已安装系统上的挂载点 |             分区              | [分区类型](https://en.wikipedia.org/wiki/GUID_Partition_Table#Partition_type_GUIDs) |                             注释                             |           建议大小           |
| :------------------: | :---------------------------: | :----------------------------------------------------------: | :----------------------------------------------------------: | :--------------------------: |
|       `/boot`1       | `/dev/*efi_system_partition*` |                          EFI System                          | [EFI 系统分区](https://wiki.archlinuxcn.org/wiki/EFI_系统分区) |            1 GiB             |
|       `[SWAP]`       |    `/dev/*swap_partition*`    |                          Linux swap                          |      [交换空间](https://wiki.archlinuxcn.org/wiki/Swap)      |          至少 4 GiB          |
|         `/`          |    `/dev/*root_partition*`    |                     Linux root (x86-64)                      |                            根目录                            | 设备剩余空间，至少 23-32 GiB |

## 格式化分区

```
# mkfs.ext4 /dev/root_partition（根分区）
```

```
# mkfs.fat -F 32 /dev/efi_system_partition（EFI 系统分区）
```

```
# mkswap /dev/swap_partition（交换空间分区）
```

## 挂载分区

```
# mount /dev/root_partition（根分区） /mnt
```

```
# mount --mkdir /dev/efi_system_partition /mnt/boot
```

```
# swapon /dev/swap_partition（交换空间分区）
```

## 生成 fstab 文件

为使需要的文件系统（如启动目录 `/boot`）在启动时被自动挂载，请使用 [genfstab(8)](https://man.archlinux.org/man/genfstab.8) 生成一个具有[持久化块设备命名](https://wiki.archlinuxcn.org/wiki/持久化块设备命名)的 [fstab](https://wiki.archlinuxcn.org/wiki/Fstab) 文件。例如，使用 `-U` 选项来通过 [UUID](https://wiki.archlinuxcn.org/wiki/UUID) 指定文件系统：

```
# genfstab -U /mnt > /mnt/etc/fstab
```

## 区域和本地化设置

程序和库如果需要本地化文本，都依赖[区域设置](https://wiki.archlinuxcn.org/wiki/Locale)，后者明确规定了地域、货币、时区日期的格式、字符排列方式和其他本地化标准。

需要设置这两个文件：`locale.gen` 与 `locale.conf`。

[编辑](https://wiki.archlinuxcn.org/wiki/编辑) `/etc/locale.gen`，然后取消掉 `en_US.UTF-8 UTF-8` 和其他需要的 UTF-8 [区域设置](https://wiki.archlinuxcn.org/wiki/Locale)前的注释（**#**）。

接着执行 `locale-gen` 以生成 locale 信息：

```
# locale-gen
```

然后创建 [locale.conf(5)](https://man.archlinux.org/man/locale.conf.5) 文件，并[编辑设定 LANG 变量](https://wiki.archlinuxcn.org/wiki/Locale#系统区域设置)，比如：

```
/etc/locale.conf
LANG=en_US.UTF-8
```

## 网络配置

为您的系统设置一个固定且易于辨识的名称（在联网环境中尤其有用），请[创建](https://wiki.archlinuxcn.org/wiki/创建) [hostname](https://wiki.archlinuxcn.org/wiki/网络配置#设置计算机名) 文件：

```
/etc/hostname
主机名
```

**提示：**请参阅 [RFC 1178](https://tools.ietf.org/html/rfc1178) 以获取一些关于为计算机取名的建议。如 [hostname(7)](https://man.archlinux.org/man/hostname.7) 所述，其必须包含 1-63 个字符，仅使用小写的 `a-z`、`0-9` 以及 `-`，但不得以 `-` 开头。

在系统内部启动网络管理器服务（开机自启）

```
systemctl enable NetworkManager
```

代理设置

```
# 设置 HTTP 代理
export http_proxy="http://代理IP:端口"
# 设置 HTTPS 代理
export https_proxy="http://代理IP:端口"
```

## 安装Grub引导

\# A. 安装 GRUB 工具包

```
pacman -S grub efibootmgr
```

\# B. 将 GRUB 安装到 EFI 分区

```
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
```

\# C. 生成启动配置文件

```
grub-mkconfig -o /boot/grub/grub.cfg
```

## 重启

输入 `exit` 或按 `Ctrl+d` 退出 chroot 环境。

可选用 `umount -R /mnt` 手动卸载被挂载的分区：这有助于发现任何“繁忙”的分区，并通过 [fuser(1)](https://man.archlinux.org/man/fuser.1) 查找原因。

最后，通过执行 `reboot` 重启系统，*systemd* 将自动卸载仍然挂载的任何分区。这时候不要忘记移除安装介质，然后使用 [root](https://wiki.archlinuxcn.org/wiki/Root) 账户登录到新系统。

# 安装Hyprland

选用Rice为caelestia-shell，https://github.com/caelestia-dots/shell

## 安装AUR

```
sudo pacman -Syu
sudo pacman -S --needed base-devel
cd /tmp
git clone https://aur.archlinux.org/paru.git
cd paru
makepkg -si
```

## 自动安装caelestia

```
paru -S caelestia-cli
caelestia install
```

## 手动Clone caelestia 仓库

```
git clone https://github.com/caelestia-dots/caelestia.git ~/.local/share/caelestia
```

```
# 1. 进入目录
cd caelestia

# 2. 安装 Hyprland 核心依赖
sudo pacman -S --needed hyprland xdg-desktop-portal-hyprland xdg-desktop-portal-gtk ttf-jetbrains-mono-nerd

# 3. 确保配置目录存在（使用 ~/.config 更稳妥）
mkdir -p ~/.config/hypr

# 4. 复制 hypr 配置文件
cp -r hypr/. ~/.config/hypr/
```
